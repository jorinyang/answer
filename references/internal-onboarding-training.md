# 内部工具上手培训方案模式

> 当 📋 新方案 领域下产出的是「内部员工工具上手培训」时，使用此模式。与 `training-proposal-template.md`（企业AI部署/DEAP工作坊）是两种不同的培训类型。

## 两种培训类型区分

| 维度 | 企业AI部署培训 | 内部工具上手培训 |
|------|--------------|----------------|
| 触发方 | 外部分客户采购AI培训服务 | 内部新员工入职，需快速补齐工具能力 |
| 受众 | 客户企业员工（混合背景，需筛选） | 本公司新员工（背景已知，但技术基础弱） |
| 内容重点 | AI平台搭建+RAG+Agent配置+场景映射 | 工具基础操作+AI使用技巧+已有Agent使用 |
| 产出结构 | 6章+附录（含报价/担忧回应/场景映射表） | 讲师逐字稿+三层目标映射+手机端实操指令 |
| 定价 | 含人天费用+差旅+物料 | 无预算项（内部培训） |
| 设备约束 | 通常有电脑 | 可能只有手机（需移动端路径） |

## 方案结构（5模块+附录）

```
模块零：开场破冰（5分钟）
模块一：工具基础操作（≈35分钟）
模块二：AI通识（≈50分钟）
模块三：Agent实战（≈55分钟）
模块四：总结与Q&A（≈35分钟）
附录：学员练习题卡
```

## 三层培训目标框架

用于从认知→理解→应用逐层定义每个模块的学习标准：

| 层次 | 定义 | 提问方式 |
|------|------|---------|
| 认知层 | "知道这是什么" | 学员能说出XX名称/列举XX项/区分XX概念？ |
| 理解层 | "理解为什么这样用" | 学员能解释为什么用XX替代旧习惯/说出XX原则？ |
| 应用层 | "能独立操作" | 学员能在手机端独立完成XX操作？ |

每个模块开篇用 callout 标注覆盖的三层目标。应用层目标必须有对应的实操练习。

## 内容规范（CRITICAL）

**核心原则：文档中的每一句话，讲师都能直接念给学员听。**

| ✅ 正确写法 | ❌ 错误写法 |
|-----------|-----------|
| "飞书和微信最大的不同是——飞书的文档，所有人能看到同一份最新版。" | "这一节的核心是让学员理解飞书和微信的区别——从微信传文件痛点切入。" |
| "来，大家打开飞书，点底部绿色的➕号。" | "🖐️停，让学员操作。手机端路径：底部➕→新建文档。" |
| "你可能听过'AI'这个词，但AI到底是什么？简单说——" | "AI概念解释：用生活类比，禁止技术术语。" |

控制标记可以保留（⏱时间/🖐️操作打断），但正文必须是可念出口的逐字稿。

## 手机端实操设计

当学员只有手机时：

- 所有飞书操作路径必须按手机端UI描述（"点底部➕"），禁止桌面端路径（"左上角新建"）
- 共享电脑（2-3台）可用于打字量大的操作（云文档编辑）
- 之客汪交互=群内@机器人，手机端天然适配
- 练习指令必须包含具体操作场景和预期结果，不能是"试着用一下"

## 内容审核时的快速修正模式

Phase 3 Architect 产出大纲后，用户可能直接指出结构问题（如"标题有歧义""缺少XX模块"）。处理方式：

1. **直接承认**，不辩护
2. **逐条修正**，修正后展示新结构
3. **不跳回前面的 Phase**——在 Architecture 阶段就地修正，继续往前
4. 如果修正涉及 Brief 层面的变更（如排除范围新增了多维表格），同步更新 Brief 记录
